# 社内ポータルアプリ 企画概要

## 1. 背景・目的

### 背景
- リモートワーク・ハイブリッド勤務の増加により、社内コミュニケーションの課題が顕在化
- 情報がSlack、Notion、Google Driveなどに散在し、ナレッジの検索性が低下
- 全社員参加のミーティングにおいて、手軽なビデオ会議ツールが不足

### 目的
1. **情報の一元管理**: 社内Wikiでナレッジを集約
2. **認証の簡素化**: GoogleアカウントでのSSO実現
3. **コミュニケーション活性化**: 社内通話ツールの提供

## 2. ターゲットユーザー

- grand2 Products 全社員
- 想定規模: 10-100名（スケーラブル設計）

## 3. 主要機能

### 3.0 実装状況注記（2026-03-10）
- 本ドキュメントは企画ベースの全体像を記載しており、実装は段階的に進行中。
- 実装済み: Wiki/Meetings/Users/Files/LiveKit基本API、管理者APIルート接続（`/api/admin`）。
- 未実装（計画先行）: Google SSO本実装、RoomモードのAI機能一式（話者分離・リアルタイム文字起こし・録画・要約生成・Wiki自動スタック）。
- 詳細な差分は `plan/implementation-status.md` を正とする。

### 3.1 認証・認可
- Google SSO (OAuth2 Proxy)
- 社内ドメイン制限 (@grand2-products.com)
- ロールベースアクセス制御

### 3.2 社内Wiki
- NotionライクなBlock-basedエディタ
- リアルタイム共同編集
- ページ階層構造
- フルテキスト検索
- 画像・ファイル添付 (Cloud Storage)
- **Room議事録の自動スタック**

### 3.3 社内通話ツール（2モード構成）

#### Everybody Coworkingモード
- 全社員の「顔」がリアルタイム更新される仮想オフィス空間
- WebRTCベース（LiveKit）による低遅延映像配信
- 在席状況の可視化（オンライン/離席中/オフライン）
- ミュート状態の表示
- ワンクリックでの即座接続

#### Roomモード
- 必要に応じて作成する会議ルーム
- ルーム作成・管理機能
- 画面共有
- **AI機能（セット）**:
  - 話者分離（Speaker Diarization）
  - リアルタイム文字起こし
  - 録画（クラウド保存）
  - AI要約・議事録生成
- 議事録は社内ポータル（Wiki）に自動スタック

## 4. インフラ構成

- **クラウドプロバイダー**: Google Cloud Platform (GCP)
- **インフラ構成管理**: Terraform
- **コンピュート**: Compute Engine (GCE)
- **コンテナオーケストレーション**: Docker Compose
- **リバースプロキシ**: Traefik (Let's Encrypt)
- **データベース**: PostgreSQL (Dockerコンテナ内)
- **ストレージ**: Cloud Storage (GCS)
- **WebRTC**: LiveKit
- **AI/ML**: Google Cloud Speech-to-Text, Vertex AI

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | ページ読み込み < 2秒 |
| 可用性 | 99.9%以上 |
| 同時接続数 | 100名対応 |
| データ保持 | 無期限（DBバックアップあり） |
| セキュリティ | HTTPS必須、SSO認証 |

## 6. 成功指標 (KPI)

1. Wiki利用率: 月間アクティブユーザー 80%以上
2. 通話ツール利用: 週1回以上の全社ミーティング実施
3. 検索成功率: 90%以上
4. Room議事録活用率: 70%以上

## 7. スコープ外

- モバイルアプリ（Web対応のみ）
- 外部ユーザー招待機能
- 高度な権限管理（部署単位など）
