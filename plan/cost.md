### 1.2 詳細内訳

#### Compute Engine

```
e2-standard-2 (2 vCPU, 8GB RAM):
- オンデマンド: $0.091/時間
- 月額 (730時間): $66.43

持続利用割引 (30日以上):
- 割引後: $46.50/月

推奨: e2-standard-2 (50人規模)
→ $50-70/月 (持続利用割引適用)
```

#### インスタンスタイプ比較

| タイプ | vCPU | RAM | 月額 (オンデマンド) | 月額 (持続割引) |
|-------|------|-----|-------------------|----------------|
| e2-small | 2 | 2GB | $19 | $13 |
| e2-medium | 2 | 4GB | $39 | $27 |
| **e2-standard-2** | 2 | 8GB | $66 | $47 |
| e2-standard-4 | 4 | 16GB | $133 | $93 |

#### Cloud Storage

```
Standard クラス (asia-northeast1):
- ストレージ: $0.023/GB/月
- 操作: Class A $0.05/1000回, Class B $0.004/1000回
- 下り: Google Cloud 内は無料

想定使用量:
- ファイル: 10GB
- ストレージ: 10GB × $0.023 = $0.23/月
- 操作: 数ドル以内
→ 合計 $1/月 程度
```

#### ネットワーク

```
下り転送 (Cloud Storage → インターネット):
- 10TBまで: $0.12/GB
- Cloud Storage から同じコンチネント: 無料

Compute Engine 下り転送:
- 10TBまで: $0.12/GB

想定使用量:
- Webトラフィック: 100GB/月
- $0.12/GB × 100GB = $12/月
```

## 5. コスト最適化戦略

### 5.1 短期 (リリース時)

- e2-medium (4GB) で開始
- Ephemeral IP 使用
- 不要なログ削除

### 5.2 中期 (50人以上)

- e2-standard-2 (8GB) にスケールアップ
- 持続利用割引適用確認
- バックアップ自動化

### 5.3 長期 (100人以上)

- e2-standard-4 (16GB) 検討
- Cloud SQL移行検討
- Cloud CDN検討

## 6. 注意点

### 6.1 隠れコスト

| 項目 | リスク | 対策 |
|------|--------|------|
| ストレージ不足 | Wiki増加で容量圧迫 | 定期アーカイブ |
| 帯域制限 | 大量転送で課金増 | Cloud Storage直接アクセス |
| バックアップ | データ損失リスク | GCSへ自動バックアップ |
| 運用工数 | サーバー管理 | 監視自動化 |

### 6.2 コスト監視

```bash
# GCE リソース使用量
gcloud compute instances describe wiki-prod --zone=asia-northeast1-a

# GCS 使用量
gsutil du -s gs://corp-internal-files-prod

# Docker リソース
docker stats
```

**予算アラート設定:**
- GCP Budgets & Alerts
- 月額 $80 超過時アラート

## 7. バックアップコスト

### 7.1 バックアップ戦略

| データ | 方法 | コスト |
|-------|------|--------|
| PostgreSQL | GCSへ日次バックアップ | ~$1/月 |
| Docker Volumes | スナップショット | ~$2/月 |
| GCSファイル | バージョニング | ~$0.5/月 |

### 7.2 自動バックアップスクリプト

```bash
#!/bin/bash
# PostgreSQL バックアップ
docker exec wiki-db-1 pg_dump -U wiki wiki > /tmp/wiki_$(date +%Y%m%d).sql
gsutil cp /tmp/wiki_*.sql gs://corp-internal-backup/postgresql/
rm /tmp/wiki_*.sql

# 古いバックアップ削除 (30日以上)
gsutil ls gs://corp-internal-backup/postgresql/ | \
  head -n -30 | xargs -r gsutil rm
```
