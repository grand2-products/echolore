# Dogfooding シナリオ一覧

各シナリオは独立して実行可能。前提条件が満たされていれば順番は問わない。

---

## 1. 初期セットアップ & アカウント

**目的:** ゼロユーザー状態から管理者アカウントが正しく作成されることを確認

- [ ] ブラウザで `http://localhost:17760` を開く
- [ ] 登録フォームが表示される（ゼロユーザー状態）
- [ ] メールアドレス + パスワードでアカウントを作成
- [ ] 作成したユーザーが admin に自動昇格している
- [ ] ログアウト → 再ログインできる
- [ ] 2人目のユーザー登録が拒否される（招待制に切り替わる）

---

## 2. Wiki 基本操作

**目的:** Wiki のCRUD と検索が正しく動作することを確認

- [ ] 新しいページを作成する
- [ ] ブロックエディタでテキスト・見出し・リスト・コードブロックを入力
- [ ] ページを保存 → 一覧に表示される
- [ ] ページを編集 → 変更が反映される
- [ ] ページを検索 → 該当ページがヒットする
- [ ] ページを削除 → 一覧から消える

---

## 3. ミーティング & LiveKit

**目的:** ミーティングルームの作成〜参加〜終了フローを確認

- [ ] ミーティングを新規作成
- [ ] ルームに参加（カメラ/マイク確認画面が出る）
- [ ] マイク ON/OFF 切り替え
- [ ] カメラ ON/OFF 切り替え
- [ ] 画面共有の開始/停止
- [ ] スタンプ/リアクションの送信
- [ ] ルームから退出
- [ ] ミーティング一覧に記録が残る

---

## 4. AI チャット

**目的:** AI エージェントとの対話が正しく動作することを確認

**前提:** `GEMINI_API_KEY` が `.env.dogfood` に設定済み

- [ ] AI チャット画面を開く
- [ ] 質問を送信 → AI が応答を返す
- [ ] Wiki の内容について質問 → 関連ページが参照される（RAG）
- [ ] 会話履歴が保持されている

---

## 5. Room AI（ミーティング要約）

**目的:** ミーティング録画 → 文字起こし → 要約 → Wiki 保存のフローを確認

**前提:** `GEMINI_API_KEY` 設定済み、LiveKit Egress 稼働中

- [ ] ミーティングルームで録画を開始
- [ ] 数分間会話する
- [ ] 録画を停止
- [ ] 要約が自動生成される
- [ ] 要約を Wiki ページとして保存できる

---

## 6. ファイル管理

**目的:** ファイルのアップロード・アクセスが正しく動作することを確認

- [ ] Wiki ページ内で画像をアップロード
- [ ] アップロードした画像が表示される
- [ ] ファイルのアクセス権限が正しい（認証必須）

---

## 7. 管理画面

**目的:** 管理者機能が正しく動作することを確認

- [ ] 管理画面にアクセスできる（admin ユーザー）
- [ ] サイト設定の変更（サイトタイトル、タグライン）
- [ ] グループの作成・メンバー管理
- [ ] ページ権限の設定
- [ ] ユーザー招待メールの送信（SMTP 設定済みの場合）
- [ ] KPI ダッシュボードの表示

---

## 8. セキュリティ

**目的:** 基本的なセキュリティ対策が機能していることを確認

- [ ] 未認証状態で API エンドポイント `/api/pages` にアクセス → 401
- [ ] 一般ユーザーで `/api/admin/*` にアクセス → 403
- [ ] CORS: 別オリジンからの fetch が拒否される
- [ ] セッション一覧でアクティブセッションが表示される
- [ ] セッションの無効化（revoke）が機能する

---

## 9. パフォーマンス & 安定性

**目的:** 本番相当の構成で致命的な問題がないことを確認

- [ ] 全サービスの healthcheck が passing
- [ ] ページ遷移が 3 秒以内に完了する
- [ ] API レスポンスが 1 秒以内に返る（単純な GET）
- [ ] 10 ページ以上作成しても一覧表示が正常
- [ ] ブラウザコンソールに致命的なエラーがない
